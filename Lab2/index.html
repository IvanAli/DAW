Conceptos sobre web

URL: Referencia a un recurso que se encuentra localizado en una red computacional.

MIME: Extensión al original protocolo SMTP para no solamente poder permitir el intercambio de texto en ASCII entre cliente y servidor, sino también datos como imágenes, audio, video.

Métodos y códigos de errores HTML:

  • Error 4xx, 5xx: Los códigos 4xx son para identificar que el error viene por parte del cliente, y el error 5xx indica que el error es por parte del servidor.
  • Bad request 400: La petición al servidor tiene sintaxis incorrecta.
  • Unauthorized 401: El parámetro de este mensaje da cuáles son los esquemas de autorización necesarios para acceder.
  • PaymentRequired 402: El parámetro de este mensaje indica cuáles son las especificaciones del esquema de cargo.
  • Forbidden 403: La petición está prohibida, sin importar la autorización.
  • Not Found 404: El servidor no ha encontrado nada en el URL especificado.
  • Internal Error 500: El servidor, por alguna razón, no pudo satisfacer la petición del cliente.
  • Not implemented 501: El servidor no soporta la facilidad requerida.
  • Service temporarily overloaded 502: El servidor no puede procesar la petición por la gran demanda de peticiones que tiene que atender.

    Preguntas

    ¿Cuál es la diferencia entre Internet y la World Wide Web?

    Internet es una masiva red de redes, que conecta millones de computadoras. La World Wide Web es la forma por la cual se puede acceder a la información que está en Internet.

    ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    • GET: Se encarga de rescatar o jalar la información de un URI (identificador del nombre de un recurso).
    • HEAD: A diferencia de GET, que tras la petición obtiene un HTTP header y body, la respuesta a HEAD no regresa un body.
    • POST: Se utiliza para que el servidor acepte la entidad que se está enviando en esta petición y sea una parte más del recurso que se identifica a través de URI. El recurso sabrá cómo manejar la entidad.
    • PUT: A diferencia de POST, que también envía una entidad a un recurso, PUT simplemente pone la entidad en el recurso (reemplazando la existente si es que había una).
    • PATCH: Solicita que una serie de cambios contenidos en la entidad por enviar sean realizados al recurso localizando en el URI.
    • DELETE: Pide que el recurso localizado en el URI sea eliminado.

    ¿Qué método HTTP se debe utilizar al enviar un formulario HTML como el de la encuesta que constestaste al principio de la actividad? ¿Por qué?

    POST, ya que se está enviando una entidad que contiene los datos que se han metido al formulario y el recurso al cual se está enviando sabrá cómo manejarlos.

    ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    GET, lo que hace es jalar la información contenida en el recurso al cual apunta el URL.

    Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    No, ya que los errores están únicamente definidos en lós codigos 4xx y 5xx, proveniendo del lado de cliente y servidor respectivamente. Un código 2xx indica éxito.

    ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    No, ya que es un error del lado del cliente (o usuario) y no del servidor.

    ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    Sí, ya que es un error del lado del servidor, y el desarrollador se encarga de que su sitio pueda ser visualizado por otros usuarios; es quien puede meter mano al servidor.

    ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    Significa que ese atributo que provenía de una versión anterior de HTML ya no se utiliza más en HTML5. Un ejemplo sería el tag 'center'; se prefiere el uso de 'text-align'

    ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

    Se agregan nuevos elementos, se deprecian algunos, y HTML5 toma más un enfoque de ser directamente para estructura del contenido de un sitio, y no su diseño.

    ¿Qué componentes de estructura y estilo tiene una tabla?

    La tabla se define con el tag 'table', dicha puede ser un encabezado 'thead', un pie 'tfoot' y un cuerpo 'tbody'. Cada renglón se define con 'tr' y cada celda con 'td'

    ¿Cuáles son los principales controles de una forma HTML5?

    legend, fieldset, input, textarea, button
